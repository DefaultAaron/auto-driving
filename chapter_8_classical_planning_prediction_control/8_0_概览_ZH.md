# 第 8 章 — 经典运动规划、预测与控制

> [!info] 状态与权威来源
> Frontmatter 中的 `status` 字段是权威来源。[[00_table_of_contents|目录]] 中的状态标记为手动同步的展示副本。

> [!abstract] 本章内容
> 把感知输出(第 7 章融合)与本体状态(第 2 章定位)变成车辆指令的经典(非 DL)栈:行为规划 + 状态机;经典预测(CV / CTRV / IMM / 社会力 / 规则);图搜索 / 采样 / 网格-样条等运动规划器;从 PID 到 MPC 的控制器;部署与控制环时序约束;安全。本章是 [[9_0_概览_ZH|第 9 章 — 深度学习规划、预测与控制]] 将要替换或混合的基线。

## 学习目标
- 为典型 AD 场景设计行为规划器与状态机。
- 实现经典预测(CV / CTRV / IMM / 社会力)并理解其相较学习方法的失败模式。
- 在 AD 相关问题上对比 A*、D*-Lite、hybrid A*、RRT / RRT*、网格 / 样条规划器。
- 调试 PID、Pure Pursuit、Stanley 控制器,理解何时需要 MPC。
- 推理实时部署约束 —— 控制环时序、ROS2 确定性通信、抖动预算、失效切换。
