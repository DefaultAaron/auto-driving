# 第 2 章 — 定位、建图与本体状态估计

> [!info] 状态与权威来源
> Frontmatter 中的 `status` 字段是权威来源。[[00_table_of_contents|目录]] 中的状态标记为手动同步的展示副本。

> [!abstract] 本章内容
> 后续每一章都要依赖的"基础设施":车辆如何知道自己在哪里、如何跨时间追踪自身状态,以及如何把状态关联到地图。包含本体状态估计、GNSS/INS/IMU 融合、激光雷达 / 相机定位、HD-map 对齐、SLAM 要点、不确定性 / 漂移 / 重定位。具体方法以**综述**层级介绍;算法深度放回对应传感器章节([[5_0_overview_ZH|第 5 章]] 的激光雷达配准、[[4_0_overview_ZH|第 4 章]] 的相机检测器)—— 本章拥有的是**概念**。

## 学习目标
- 解释本体状态、世界 / 车辆 / 传感器坐标系与时间对齐为何是后续一切的基础。
- 通过 EKF / UKF / 因子图融合 GNSS、INS、IMU 与里程计得到稳定的本体状态。
- 在综述层面推理何时用激光雷达定位、何时用相机定位。
- 把本体位姿对齐到 HD-map,推理新鲜度、漂移与变更检测。
- 读懂现代 SLAM 文献(LIO-SAM、FAST-LIO、ORB-SLAM3 谱系),在线 vs 离线建图如何选。
- 诊断常见失效模式:GNSS 拒止、城市峡谷、传感器退化、标定漂移。
